# AI Agent System for Go Backend Development

## Overview

This project contains an AI multi-agent system for Go backend development with 5 specialized agents coordinated by a Workflow Orchestrator.

## Agent Commands

```
/agent-full "<requirement>"   - Full pipeline: Plan -> Code -> Test -> Review
/agent-plan "<requirement>"   - Design architecture and create plan
/agent-code                   - Generate code from plan
/agent-test                   - Generate and run tests
/agent-fix "<error>"          - Debug and fix bugs
/agent-review                 - Review code quality and security
```

## Rules for ALL Agents

1. **ALWAYS read `.rules/*` before starting any task**
2. **NEVER auto-commit or push code** - user must explicitly request
3. **NEVER merge to main branch** - work on temp branches
4. **Follow Go idioms** - "Accept interfaces, return structs"
5. **Consumer-side interfaces** - define interfaces at the caller, not the implementor
6. **Minimal changes** - don't over-engineer, don't add unrequested features
7. **Security first** - validate input, parameterize queries, no hardcoded secrets

## Go Backend Conventions

- Project layout: `cmd/`, `internal/`, `pkg/`
- Dependency direction: handler -> service -> domain (strict)
- Error handling: `fmt.Errorf("%w", err)`, `errors.Is/As`
- Context: first parameter in service/repository methods
- Tests: table-driven with `testify/assert` + `gomock`

## Workflow State Machine

```
IDLE -> PLANNING -> CODING -> TESTING -> REVIEWING -> DONE
                                            |
                                          FIXING (max 3 loops)
                                            |
                                          ESCALATE (ask user)
```
